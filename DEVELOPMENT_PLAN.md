# 建筑安全智能监控平台开发计划

## 目标与范围

面向智慧工地监管员和设备管理员，交付一套可在本机完整启动、使用确定性演示数据验收的系统。塔吊、环境、升降机、高支模和深基坑共用“设备上报 → 数据校验与幂等入库 → 阈值规则判断 → 告警合并 → 实时通知 → 人工处置 → 审计追踪”主链路。项目以 `D:\building-agent\platform` 为主工作目录，并包含只读 Agent、受控知识/报告审核和外部集成中心。

## 素材审计结论

- 前端初始包包含 Vue 3 依赖和大量页面图片，但只有登录页源代码，其余只有构建产物，不能继续维护。
- 服务端初始包只有公共类、异常类和配置片段，没有 Maven 工程、控制器、服务或持久化实现。
- `db_build_safe.sql` 提供了原业务表和样例数据，用于对齐字段语义；本项目另建可迁移的精简领域模型。
- `tower_crane_data.csv` 用作设备模拟器和趋势测试的确定性输入。
- `tdcfv2.glb` 用作塔吊 Three.js 参数绑定；`last.pt` 由 AI 适配服务保留为可选模型资源。
- 素材的运行时使用、仅参考、未采用和禁止交付边界以 `docs/MATERIAL_USAGE_AUDIT.md` 为准；未验证模型的启用条件见 `docs/AI_MODEL_MANIFEST.md`。

## 技术决策

- 前端使用 Vue 3、Vue Router、Vuex、ECharts、Three.js；保持单一状态管理方案。
- 后端使用 Java 17+、Spring Boot 3.3、Spring Security、JDBC、WebSocket；默认 H2 便于验收，可切换 MySQL 8。
- 设备 TCP 采用 4 字节大端长度 + UTF-8 JSON 帧，包含消息编号、协议版本、设备编号、采集时间和指标数组。
- 视频、视觉 AI 和喷淋外部基础设施并未随素材提供，因此实现精确白名单、健康/传输探测、网络契约测试和人工复核闭环；未提供真实端点时保持 `NOT_CONFIGURED/SIMULATED`，不伪装成生产可用。
- AI Agent 默认使用明确标注的 `DEMO` 工地摘要回答；外部模式只追加当前工地已人工审核发布的受限知识摘录。API Key 不进入浏览器，Agent 不执行设备或业务写操作。
- 演示数据使用固定值、固定时间轴和稳定消息 ID，并受 `DEMO_DATA_ENABLED` 控制，不通过随机数制造实时效果；聚合页面通过 `dataLabel` 标识数据性质。

## 完成定义

- 关键接口具备参数校验、服务端权限校验、明确错误码和追踪标识。
- 重复遥测不会重复入库或生成重复告警；同规则告警按抑制窗口合并。
- 告警只允许按“待确认 → 处理中 → 已解决 → 已关闭”前进，每一步生成处置记录和审计日志。
- 设备、AI、视频、喷淋均具有正常、空、离线或失败状态。
- 当前自动化测试和前端构建须持续保持通过；PC 端 E2E、视觉复验与 MySQL 8 冒烟完成后，才满足最终完成定义。

- 完成设备编辑；新增规则支持作用域选择与指标/来源/设备类型校验，已有规则仍按原作用域编辑。

### 第三轮（已完成）

- 已完善喷淋失败回执、任务超时、最短执行间隔、设备绑定与在线校验，以及页面演示回执。
- 已补齐告警区域/时间筛选和 CSV 导出，并限制管理员用户与审计日志只能访问授权工地。
- 第三轮结束时自动化基线为前端 21 项、后端 44 项，均通过；PC 端生产构建通过。

### 迁移与 AI Agent 扩展（已完成）

- 项目已完整迁移到 `D:\building-agent`，关键配置、开发要点和构建入口校验一致；原 C 盘副本暂留作回滚，不作为后续开发主目录。
- 已逐项复核用户 ZIP 与开发要点：塔吊 CSV、`tdcfv2.glb` 和可选 `last.pt` 已按必要边界使用；旧 SQL、旧 Apifox、未使用字体和图 19–21 需求证据已分类隔离或记录。
- 新增第十二个 PC 模块“AI Agent 问答”，包含会话与消息服务端分页、新会话、快捷问题、完整等待/错误状态，以及用户与工地双重隔离。
- 后端支持 `DEMO`、`OPENAI_COMPATIBLE`、`DISABLED` 三种模式；外部 API 的地址、白名单、模型和密钥均由服务端环境变量管理，上游失败不会留下半条问答记录。远程地址仅允许 HTTPS，并具备单实例会话顺序锁、每用户限流、并发舱和上游响应读取上限。

### 第四轮（已完成）

- 已在独立 MySQL 8.0.34 临时实例验证 19 表空库初始化、DDL 二次执行、InnoDB、治理工作流及完整业务闭环；临时库、账号、端口和数据目录在测试后清理。
