# 已知限制

以下内容是当前 1.0.0 实训交付的明确边界。它们不影响本地 PC 演示，但在共享测试、长期运行或生产部署前需要处理。

## 1. PC 前端

- 本次只以桌面浏览器和不低于 `1366 × 768` 的视口验收；移动端、触控交互和小屏信息架构不在承诺范围内。
- `start.ps1` 启动的是 Vite 开发服务器，适合本地演示，不是生产静态服务器。
- 规则、摄像头和喷淋任务接口已提供服务端分页和页面所需筛选；区域场景、工地选择器等基础字典列表仍一次完整返回。
- 塔吊与大型设备选择均通过服务端分页读取；区域场景和工地选择器等基础字典列表仍没有分页。
- 前端 API 文档由人工维护，没有从 OpenAPI 契约自动生成，接口变化时需同步更新。

## 2. 演示数据与统计

- 默认 H2 演示环境首次空库会创建固定工地、12 台设备、规则、告警、摄像头、AI 风险和遥测数据；这不是实时生产数据。MySQL 模板默认关闭该初始化。
- 综合首页的设备、在线、告警、待复核风险、七日风险趋势和统计时点均从当前数据库计算；`dataLabel` 仍会按配置明确区分演示与业务数据。
- 环境摘要的最新指标、超标记录和统计时点均来自当前数据库；环境超标列表当前最多返回 10 条，尚未提供继续翻页。
- `DataInitializer` 仅在 `DEMO_DATA_ENABLED=true` 时运行，并以 `app_user` 是否非空判断是否整体初始化；大型设备与扩展塔吊遥测有独立的幂等补齐器，但存在任意用户时仍不会补齐缺少的其他基础业务表。

## 3. 身份与权限

- Token 只保存在单个后端进程内存中，后端重启即失效，无法多实例共享。后端虽提供会立即撤销旧 Token 的刷新接口，但 PC 端尚未实现到期前自动刷新。
- 演示账号和初始口令写在初始化代码中；管理员可新增、启停和重置密码，但没有用户编辑/删除、自助改密、锁定、MFA、登录限流或密码过期策略。
- WebSocket Token 位于查询字符串，可能出现在代理访问日志；生产应改为更安全的握手认证方式。
- 审计日志已按所选授权工地过滤，但历史表本身没有 `site_id` 快照；当前根据关联对象的现有归属或操作者范围推导，关联对象日后跨工地迁移时不能还原操作发生时的原始归属。日志仍没有导出、归档、不可篡改存储或保留期策略。

## 4. 设备接入与实时性

- TCP 9100 没有 TLS、设备证书、签名、重放窗口或每设备密钥；仅适合受控本地网络，不能暴露到不可信网络。
- TCP 监听使用通用服务器套接字，默认可能绑定所有网卡；不需要时应设置 `TCP_ENABLED=false` 并用防火墙限制。
- REST 遥测写入有角色鉴权，但 TCP 遥测不使用用户角色，设备身份仅依赖 `deviceCode`。
- WebSocket 是尽力投递，没有消息队列、持久游标、重放、确认或断线补偿；前端断线后只能重新读取 REST 数据。
- 已提供可配置的心跳超时调度、原子离线转换、`SYSTEM_DEVICE_OFFLINE` 去重告警和遥测恢复；固定 H2 演示快照默认关闭该调度，接入持续数据源后必须显式启用。多实例部署仍建议增加分布式调度锁与运行指标。

## 5. 规则与告警

- `SITE/TYPE/DEVICE` 规则会校验工地、指标来源和目标设备类型；塔吊、环境、升降机、高支模和深基坑均已接入自动告警。当前表结构未独立保存 TYPE 目标类型，因此同时属于高支模与深基坑的共享指标必须使用 DEVICE 范围，不能创建 TYPE 规则。
- 规则表达式只支持单指标和五种比较运算，没有组合条件、持续时间、滑动窗口或跨设备关联。
- 抑制窗口只合并仍未关闭、同设备同规则的近期告警，没有分布式锁；多后端实例并发时需要数据库级并发设计。
- 大型设备默认规则只在 `DEMO_DATA_ENABLED=true` 时幂等补齐，不代替真实项目的专项监测方案和现场阈值复核。
- 告警只允许单向状态机，没有重新打开、撤销、转派、附件或 SLA。
- 遥测与环境趋势已返回最近的有限数据点；长期数据仍需要时间范围、游标、聚合和降采样，当前最多返回 500 个原始点。

## 6. 视频与 AI

- 平台已提供受主机白名单保护的视频地址配置和 HTTP/RTSP/RTMP 传输探测，但演示库未提供真实流；探测成功不等于浏览器已完成解码、鉴权、转码、录像或兼容性验收。
- 提供的 `last.pt` 未附训练标签说明、验证集、精度报告、硬件基准或运行时锁定，因此默认不加载。
- `AI_DEMO_MODE=true` 返回固定样例并明确标记，不代表真实模型推理。
- 主业务已通过受白名单、超时和有界响应保护的桥接调用独立 AI 适配器，并把合法检测写入待人工复核风险；默认仍为关闭状态，现有 `last.pt` 尚未通过质量验收。
- 独立 AI 服务自身仍没有 TLS、任务队列、并发限流或 GPU 资源隔离，只应位于受控内网并由主业务层鉴权。

## 7. AI Agent 问答

- 主业务 Agent 默认运行在 `DEMO`，回答由当前工地数据库聚合快照确定性生成并明确标注演示模式，不调用或代表已经验收的真实大模型。`DISABLED` 可完全停用问答。
- `OPENAI_COMPATIBLE` 只支持兼容 `/chat/completions` 的非流式完整响应；没有流式增量、工具调用、函数调用、结构化输出约束、重试队列或服务商故障切换，真实模型质量、延迟、费用和安全性尚未验收。
- Agent 读取固定工地摘要以及与问题匹配的最多 3 个已发布知识摘录；不包含任意原始表访问，也不能执行告警处置、设备控制、规则修改、喷淋下发或其他写操作。
- 启用外部模式后，服务端会把当前问题、受数量上限约束的本会话历史，以及上述字段范围内的工地摘要发送给所配置服务商。个人 API Key 只写入后端并以 AES-256-GCM 加密，远程地址强制 HTTPS，且 Base URL 受管理员精确白名单限制；部署方仍须自行完成服务商审批、数据最小化、个人信息处理、保留策略和跨境合规评估。
- 个人凭据当前使用一个部署级主密钥，没有密钥版本表或在线轮换/重新加密流程；主密钥丢失会导致已有密文不可恢复，替换主密钥会使旧配置无法解密，生产部署必须把数据库备份与主密钥分离保管并制定轮换方案。
- 带有界等待的会话条带锁、外部请求并发舱壁和每用户分钟限额都只保存在单个后端实例的内存中。多实例部署无法跨实例保证同一会话的请求顺序，也不能形成全局用户配额或全局并发上限；至少需要按用户/会话设计粘性路由，或用 Redis/数据库实现集中式会话锁与限流。仅使用粘性路由仍不能把每实例舱壁变成全局并发上限。
- 会话已按“当前用户 + 工地”隔离并支持会话/消息分页，但没有重命名、删除、导出、归档或保留期限管理；数据库中的消息会一直保留到运维人员按正式治理流程处理。
- 已实现知识草稿/提交/审核/发布/归档、受控检索、报告模板/生成/人工审核，以及 LOG/Webhook 投递记录；当前没有二进制批量导入、向量数据库、复杂编排、邮件/企业 IM 专用连接器、审核会签或电子签章。

## 8. 喷淋控制

- 喷淋支持 `DEMO/HTTP/DISABLED` 三种网关模式、精确地址白名单和认证回调；默认仍为 `DEMO`，未提供真实 PLC/IoT 端点时不能证明现场设备实际启停。
- 服务端已提供同区域最短任务间隔、创建与下发双重设备可用性校验，以及 `DISPATCHED` 回执超时自动转 `FAILED`；这些配置只保护平台任务状态，不能证明现场设备实际启停。
- 超时扫描使用数据库条件更新保证单条任务只完成一次状态转换，但没有分布式调度锁；多实例会重复扫描候选行，只有成功更新的实例写审计和事件。
- 平台命令编号可供设备侧实现幂等，但尚无命令签名、自动重试、用水量、天气联锁、急停、审批或现场安全互锁。

## 9. 数据库与运维

- H2 适合本地验收，不适合多实例生产并发。
- MySQL 结构通过 `CREATE TABLE IF NOT EXISTS` 初始化，没有 Flyway/Liquibase 版本迁移和回滚脚本。
- H2 与 MySQL 之间没有自动数据迁移工具；原始 `database/reference/db_build_safe.sql` 与当前精简领域模型不兼容。
- 数据库连接没有连接池专项调优、只读副本、分区、归档和自动备份策略。
- 启动脚本的进程停止依赖操作系统终止请求；在关闭或复制 H2 文件前，应确认后端进程已经退出。
- 已暴露受保护的 Prometheus 指标和 Actuator 健康检查；本地日志仍没有轮转、集中采集、长期指标存储或生产告警策略。

## 10. 测试覆盖

- 前端已有单元测试与可重复执行的 Playwright PC E2E；E2E 覆盖登录、分页、塔吊 WebGL、告警和 Agent 主路径，但尚未形成多浏览器兼容、视觉差异基线或无障碍认证。
- 后端 22 个默认测试套件、79 项测试覆盖原有主链路以及知识/报告审核、受控投递、个人 Agent 凭据加密/隔离、集成状态、外部网络契约和 Actuator 权限；独立 MySQL 8 冒烟另有 2 项。尚缺真实外部模型/视频/网关硬件、跨实例并发和性能专项测试。
- 模拟器测试集中在协议纯函数，没有真实网络异常、慢客户端和长时间稳定性测试。
- AI 测试只验证默认安全模式，没有验证模型精度、性能或资源上限。
- 尚未完成高并发、容量、故障注入、安全扫描、依赖漏洞、浏览器兼容和无障碍专项认证。

## 11. 生产化优先级

建议依次完成：

1. TCP 设备认证、WebSocket 更安全的握手凭据与全链路 TLS；
2. 外部持久会话、完善的用户与密码生命周期、登录保护；
3. Flyway/Liquibase、数据库备份恢复和正式数据迁移；
4. 将现有 PC E2E 与独立 MySQL 冒烟接入 CI 门禁；
5. 精确 `TYPE` 规则、长期趋势游标与降采样、基础字典列表容量治理；
6. 使用实际视频、模型和喷淋硬件完成现场专项验收；
7. Agent 外部服务商与数据合规验收、会话生命周期治理，以及邮件/企业 IM 专用推送；
8. 生产静态托管、服务守护、日志集中化与指标告警。
