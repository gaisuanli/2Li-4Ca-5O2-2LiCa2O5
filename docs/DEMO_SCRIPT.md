# PC 端演示脚本

## 1. 演示目标

在约 20 分钟内展示十四个 PC 模块、服务端分页、设备数据接入、规则告警、AI 人工复核、AI Agent 演示问答、受控知识与报告治理、外部集成验收、喷淋控制、管理员用户管理和审计闭环。所有固定内容均明确标注为演示数据，不宣称已接入真实摄像头或验证过的 AI 模型。

## 2. 演示前准备

建议使用 `1440 × 900`、浏览器缩放 100%。从项目根目录执行：

```powershell
.\scripts\build.ps1
.\scripts\start.ps1 -WithSimulator -WithAi
```

预先打开：

- 管理端：<http://127.0.0.1:5173>
- 设备模拟器：<http://127.0.0.1:9200>
- 后端健康检查：<http://127.0.0.1:8080/api/health>
- AI 健康检查：<http://127.0.0.1:5001/health>

首次演示建议使用全新的数据库 URL，避免上次演示已经复核风险或关闭告警。例如复制 `.env.example` 为 `.env` 后，将 `DB_URL` 中的文件名改成 `sitesafe-demo-01`。不要在进程运行时删除 H2 文件。

## 3. 演示流程

### 0:00–1:00 健康与边界

1. 展示后端健康检查中服务和数据库均为 `UP`。
2. 展示 AI 健康检查默认 `mode=DISABLED`，说明权重文件存在不等于模型已验证。
3. 登录后打开 `/api/agent/config`，确认主业务 Agent 为 `mode=DEMO`、`available=true`；说明这只启用基于数据库聚合快照的演示回答，不是外部真实模型。
4. 说明本次只验收 PC 管理端，左侧十四个页面分别对应独立业务模块。

### 1:00–2:30 登录与权限

1. 使用 `supervisor / Safe@123` 登录。
2. 指出顶部工地、用户角色和实时通道状态。
3. 指出 supervisor 可见规则配置，但看不到仅 ADMIN 可见的用户管理和审计日志。

讲解要点：前端菜单按角色收敛，后端仍会独立执行角色和工地范围校验。

### 2:30–4:00 综合首页

1. 展示设备总数、在线率、未闭环告警和待复核风险。
2. 展示告警等级、设备类型、最近告警和趋势图。
3. 指出统计时点旁的“演示数据”标识。

### 4:00–5:30 设备模块与分页

1. 进入“设备总览”，点击工地区域场景筛选设备。
2. 使用关键字、设备类型或连接状态筛选。
3. 清除筛选，确认 12 台种子设备在每页 10 条时分为两页。
4. 点击“下一页”“首页”，再切换每页 20 条，说明查询条件、总数、页码和页容量都由服务端控制。
5. 选择一台设备，展示区域、位置、连接和启用状态的职责分离。

### 5:30–7:00 塔吊与 TCP 数据

1. 打开模拟器，点击“连接 TCP”，连续点击“发送一次”可观察下一设备在两台塔吊、升降机、高支模和深基坑之间轮询。
2. 确认发送数和 ACK 数增加；ACK 包含新增/重复指标数量。
3. 返回“塔吊分析”，选择 1 号塔吊，展示三维模型与风速、吊重趋势；回转角、吊钩高度和工作幅度分别绑定模型节点。
4. 可开启自动发送观察至少五个周期，再关闭，避免演示结束后持续写数据。

讲解要点：CSV 与内置大型设备档案被转换成协议 1.0 遥测；消息编号与指标联合唯一，重复上报不会重复入库。

### 7:00–8:00 大型设备监测

1. 打开“大型设备监测”，依次切换升降机、高支模和深基坑。
2. 展示当前指标、指标切换和最近 48 点趋势。
3. 开启模拟器自动发送，等待对应设备上报超过默认阈值的数据，在告警中心确认产生或合并告警。
4. 选择原离线的深基坑设备，说明接收到有效遥测后会恢复在线；页面时间仍以最近持久化采集时间为准。

### 8:00–9:30 环境与喷淋

1. 进入“环境分析”，切换环境站和 PM2.5/噪声趋势。
2. 选择关联在线喷淋设备的东区环境监测站。
3. 填写原因并创建喷淋任务，此时状态为“已创建”。
4. 点击“下发指令”，展示生成命令编号和“已下发”。
5. 点击“记录演示成功回执”，状态变为“已执行”。
6. 选择另一条已下发演示任务，填写失败原因并点击“记录演示失败回执”，状态变为“失败”；说明空原因会被服务端拒绝。

强调当前回执是明确标记的演示动作，不代表真实喷淋网关已接入；服务端还会校验区域最短间隔、设备绑定与在线状态，并将长期未回执任务自动标记失败。

### 9:30–11:00 AI 风险人工复核

1. 进入“AI 风险”，筛选“待复核”。
2. 选择“未佩戴安全帽”记录，展示模型版本、置信度、摄像头和区域。
3. 填写“现场负责人确认人员未佩戴安全帽”并点击“确认有效”。
4. 记录返回后的状态变化，说明确认有效才会生成高等级告警。

不要重复复核同一记录；服务端会以 `409 RISK_ALREADY_REVIEWED` 拒绝。

### 11:00–12:30 AI Agent 演示问答

1. 进入“AI Agent 问答”，确认页面标明 `DEMO` 模式，并说明浏览器中没有服务商 API Key。
2. 新建会话，使用快捷问题或输入“汇总当前工地尚未闭环的告警，并给出处置优先级”。
3. 发送后确认同时出现用户消息和带“演示模式”标识的助手回答；快照只包含工地名称、区域数、设备总数与启用设备在线/离线、活动告警、待复核风险、喷淋任务和最近遥测时间等聚合值，不包含具体环境指标。
4. 再问“当前有哪些待复核 AI 风险？”，展示同一会话的上下文和消息分页；切换会话列表分页后仍只显示当前登录用户、当前工地的会话。
5. 切换工地或换一个账号，说明服务端同时按会话所有者和工地范围隔离，其他用户不能读取该会话。

不要把 DEMO 回答描述为真实大模型推理。若课堂另行配置 `OPENAI_COMPATIBLE`，只演示已经完成安全审批的服务商；外部模式的连通不代表模型质量已经验收。

### 12:30–14:30 告警处置闭环

1. 进入“告警中心”，按来源 `AI_RISK` 或高等级筛选刚创建的告警。
2. 展示列表分页、详情、重复次数和处置时间线。
3. 选择区域和开始/结束时间，确认列表仅保留筛选范围内告警；点击“导出 CSV”，核对下载文件沿用相同条件。
4. 依次填写备注并执行“确认处理”“标记解决”“关闭告警”。
5. 指出状态只能按 `PENDING → PROCESSING → RESOLVED → CLOSED` 前进，不能越级或回退。

### 14:30–16:00 规则配置

1. 进入“规则配置”，展示指标、运算符、阈值、等级、范围和抑制秒数。
2. 新增“塔吊回转角预警”，或只修改一条现有规则后保存。
3. 停用再启用该规则，说明停用不会删除历史告警。

讲解要点：规则由服务端保存并在遥测事务中执行，抑制窗口用于合并重复告警。

### 16:00–18:00 管理员用户管理

1. 退出 supervisor，使用 `admin / Admin@123` 登录。
2. 打开“用户管理”，展示用户名/姓名搜索、总数及 10/20/50 条服务端分页。
3. 新建一个监管员账号，选择角色与工地范围；说明密码至少 8 位且页面不会回显。
4. 演示重置密码或启停账号，指出操作会使目标用户已有会话立即失效，且管理员不能停用自身。
5. 说明管理员不能查看、创建或修改超出自身授权工地范围的账号，服务端会返回 `SITE_SCOPE_DENIED`。

### 18:00–19:30 审计追踪

1. 打开“审计日志”，确认页面携带当前工地，并按每页 20 条查看刚才的用户管理、风险复核、AI Agent 会话/消息、告警处置、喷淋和规则操作。
2. 展示每项的用户、对象、说明和 `traceId`。
3. 切换分页容量，说明审计列表同样使用服务端分页；切换授权工地后列表重新加载且不会混入其他工地记录。

## 4. 角色补充演示

如时间允许，使用 `device / Device@123` 登录：

- 可新增、编辑或启停设备；
- 可控制喷淋；
- 看不到规则、用户管理与审计页面；
- 调用告警处置接口会被后端返回 `403`。

## 5. 异常场景备选

| 场景 | 演示方式 | 预期表现 |
| --- | --- | --- |
| 后端未启动 | 先开前端 | 页面显示明确加载失败与重试，不伪造数据 |
| 模拟器未连接 | 点击发送 | 显示“TCP 尚未连接” |
| 设备停用 | 设备管理员停用后上报 | 返回 `DEVICE_DISABLED` |
| 重复遥测 | 重复发送同一消息 | `duplicateMetrics` 增加，不重复入库 |
| 告警越级 | 对 PENDING 直接 RESOLVE | 返回 `INVALID_ALARM_TRANSITION` |
| AI 重复复核 | 再次复核同一风险 | 返回 `RISK_ALREADY_REVIEWED` |
| Agent 停用 | 设置 `AI_AGENT_MODE=DISABLED` 并重启后发送问题 | 返回 `AI_AGENT_DISABLED`，不会写入半轮消息 |
| Agent 请求过频 | 在隔离环境降低每用户分钟配额后连续发送 | 返回 `AI_AGENT_RATE_LIMITED`，不会写入半轮消息 |
| Agent 繁忙 | 在隔离环境降低并发/锁等待上限并制造并发 | 返回 `AI_AGENT_BUSY`，页面可稍后重试 |
| 视频未配置 | 打开“视频监控” | 显示 `NOT_CONFIGURED`，没有伪造视频 |

## 6. 演示结束

1. 在模拟器关闭自动发送并断开 TCP。
2. 退出登录。
3. 执行：

```powershell
.\scripts\stop.ps1
```

4. 保留 `runtime/logs/`、关键页面截图以及告警与审计记录作为验收证据。
