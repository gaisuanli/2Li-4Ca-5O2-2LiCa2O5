## 3. 反序列化安全风险

PyTorch `.pt` 文件常通过 Python Pickle 机制或与其等价的对象反序列化路径加载。反序列化不可信文件可能执行文件中构造的任意 Python 代码，进而读取密钥、修改文件或控制运行进程。即便文件哈希与本清单一致，也只能说明它与当前收到的文件相同，不能说明它安全。

因此：

- 不得在开发人员主机、带云凭据的环境或生产服务器上直接“试加载”；
- 不得因为文件来自课程压缩包就将其视为可信；
- 安全评审应优先采用不触发任意对象构造的检查手段；如果工具链允许，应在受控环境验证并转换为更安全、结构明确的部署格式；
- 任何首次反序列化都必须放在无业务数据、无密钥、无出站网络、非特权用户和可销毁文件系统的隔离环境中。

## 4. 启用前的强制条件

只有以下条件全部完成并形成记录后，才可把 `AI_ENABLE_MODEL` 设置为 `true`：

1. 明确模型上游来源、作者、版本和许可，确认允许当前用途与再分发。
2. 在一次性隔离环境中完成安全分析、首次加载和依赖兼容性验证；运行账户没有管理员权限、业务凭据和不必要的网络访问。
3. 固定并审核 Python、Ultralytics、PyTorch、Pillow 及底层运行库的精确版本和哈希，而不是只依赖宽泛版本范围。
4. 建立经过业务确认的类别表、输入预处理、置信度阈值、框坐标和风险类型映射。
5. 使用独立、可复现且合法的数据集评估误报、漏报、延迟、资源峰值和典型恶劣场景；保存版本化报告。
6. 对上传图片实施真实格式解码、像素尺寸/文件大小限制、资源配额、并发限制、超时和异常隔离。
7. 对 AI 服务增加认证、TLS 或可信内网边界、审计、监控和最小权限部署，不把当前本地适配器直接暴露到公网。
8. 保留人工复核环节；模型输出不得直接关闭告警、操作设备或替代安全责任人的判断。

若任何一项没有完成，应保持模型模式关闭。`AI_DEMO_MODE=true` 只返回明确标记的固定样例，不能作为模型验证证据。

## 5. 与 AI Agent 的边界

`last.pt` 属于本地视觉检测适配器；侧边栏的 AI Agent 问答使用服务端配置的大模型提供方。两者拥有不同的输入、依赖、安全风险和验证流程：

- 启用 AI Agent API 不会验证或自动启用 `last.pt`；
- 启用 `last.pt` 也不会给 Agent 提供知识库、报告生成或推送能力；
- 外部大模型 API 还需单独评估会话内容和工地汇总数据的传输、保存、配额与隐私边界。

素材采用总表见[实训素材使用审计](MATERIAL_USAGE_AUDIT.md)。
